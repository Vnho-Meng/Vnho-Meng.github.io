2.函数的参数

可以使用函数的参数实现不同的代码

作用: 在函数内部某些值不能固定,可以用 参数 传递进不同的值

  • 形参和实参

    • 在声明小括号内的参数是形参(形式上的参数)
    • 在函数调用的函数里是实参(实际的参数)
    • 形参是接收实参的,类似于一个不用声明的变量
    • 参数可以有无限个或没有

  • 形参和实参的匹配

    1. 如果实参个数和形参个数一直,则正常输出结果

    2. 如果实参个数多于形参个数,只会取到形参个数

    3. 如果实参的个数小于形参的个数,传入的参数是undefined

3.函数的返回值

  • 函数只是实现某种功能,最终返回结果需要返还给函数 调用者:函数名()

  • 只要函数遇见 return,就把后面的结果返回给函数调用者

    • return 后面的代码不会被执行
    • return 只能返回一个值
    • 函数如果有 return ,返回的是 return 的值,如果函数没有return ,返回的是undefined

    • splice() 方法通过删除或替换现有元素或者原地添加新的元素来修改数组,并以数组形式返回被修改的内容。此方法会 改变原数组。

    • 由 被删除的元素 组成的一个 新数组 。如果只删除了一个元素，则返回只包含一个元素的数组。如果没有删除元素，则返回空数组。

      • start: 指定修改的开始位置（从0计数）。如果超出了数组的长度，则从数组末尾开始添加内容；如果是负值，则表示从数组末位开始的第几位（从-1计数，这意味着-n是倒数第n个元素并且等价于 array.length-n ）；如果负数的绝对值大于数组的长度，则表示开始位置为第0位。

      • deleteCount: 可选, 整数, 表示要移除的数组元素的个数。如果 deleteCount 大于 start 之后的元素的总数，则从 start 后面的元素都将被删除（含第 start 位）。

9.DOM 事件流

  • 事件流描述的是从页面接收事件的顺序

  • 事件发生时,毁在元素节点之间按照特定的顺序传播,这个传播过程就是 DOM 事件流

    ![](W:\Note\images\DOM 事件流.png)

  • 分为三个阶段

    1. 捕获阶段
    2. 当前目标阶段
    3. 冒泡阶段

  • 事件捕获: 网景公司提出 , 由 DOM 最顶层节点开始 , 然后逐级向下传播到最具体的元素接受过程

  • 事件冒泡: IE公司提出 , 时间开始时由最具体的元素接受,然后逐级向上传播到 DOM 最顶层节点

  • 在实际开发中喜欢用冒泡阶段来解决事件注册
  • 一些事件没有冒泡: onblur / onfocus / onmouseeter / onmousleave

let

  • 在es6中新增的变量声明方法let

  • 这个方法声明的变量具有块级作用域,也就是说在其他作用域内访问不到这个变量

  • 防止循环体内的变量成为全局变量

  • 使用 let 关键字声明的变量没有变量提升

const

  • 声明常量,常量就是值(内存地址),定以后内部值不可变化
  • 也具有块级作用域
  • 声明常量时,必须给予一个初始值,否则会报错
  • 简单数据类型不可直接更改,复杂数据类型可以更改其内部的值,但是不能直接更改整个复杂数据类型的值

var let const 的区别

  1. 使用 var 声明的变量,其作用域为 该语句所在的函数内,且存在变量提升
  2. 使用 let 声明的变量,其作用域为 该语句所在的代码块内,且不存在变量提升
  3. 使用 const 声明的变量,在后面出现的代码块中, 不能修改该常量的值

6. node 模块加载机制

  3. 如果找到了同名文件夹,会在文件夹中找 index.js 文件
  4. 如果文件夹中没有 index.js 文件,就会去当前文件夹中的 package.json 文件中查找 main 选项的入口文件
  5. 如果找指定入口文件不存在或没有指定入口文件,就会报错
  1. 当模块没有路径且没有后缀时 node.js 会假设它是系统模块
  2. nodejs 会去 node_modules 文件夹中查找
  3. 首先看是否有同名 js文件 如果找到了就执行
  4. 没找到就再看是否有改名字的文件夹
  5. 在文件夹中找 index.js
  6. 如果没有 index.js 就查看该文件夹下的 package.json 文件中的 main 选项 确定模块入口文件
  7. 否则找不到——-报错
